Thread Lifecycle & Creation

5.1 🟢 Basic

What are the states in the Java thread lifecycle?

Why asked: Tests foundational understanding of thread state machine — essential before deeper concurrency topics.
💡 Hint

Java defines 6 states in Thread.State: NEW, RUNNABLE, BLOCKED, WAITING, TIMED_WAITING, TERMINATED. A thread transitions from NEW → RUNNABLE on start(), enters BLOCKED when waiting for a monitor lock, WAITING via wait()/join(), and TIMED_WAITING via sleep(ms) / wait(ms).

5.2 🟢 Basic

What is the difference between extending Thread, implementing Runnable, and implementing Callable?

Why asked: Common starter question to gauge OOP understanding and awareness of return values / checked exceptions.
💡 Hint

Thread extension is limiting (single inheritance). Runnable has no return value and cannot throw checked exceptions. Callable<V> returns a Future<V> and can throw checked exceptions — preferred with ExecutorService.submit().

5.5 🟡 Intermediate

What is Thread.interrupt() and how do you handle interruption correctly?

💡 Hint

Interruption sets an interrupt flag. Blocking methods like sleep(), wait(), join() throw InterruptedException when interrupted. In a catch block you should either re-interrupt (Thread.currentThread().interrupt()) or rethrow, never silently swallow.

synchronized, volatile & Happens-Before

5.6 🟡 Intermediate

Explain the Java Memory Model (JMM) and the happens-before relationship.

Why asked: Senior-level question — the JMM underpins all correct concurrent code; misunderstanding leads to subtle bugs.
💡 Hint

Happens-before is a guarantee: if action A happens-before action B, A's effects are visible to B. Sources: synchronized blocks, volatile writes/reads, Thread.start(), Thread.join(), and Future.get(). Without happens-before, the CPU or compiler may reorder operations.

Why asked: Reveals understanding of monitor locks — instance lock vs. class-level lock.
5.8 🟡 Intermediate

What does the volatile keyword guarantee and what does it NOT guarantee?

💡 Hint

volatile guarantees visibility (reads always from main memory) and prevents instruction reordering around that variable. It does NOT guarantee atomicity for compound actions like i++ (read-modify-write). Use AtomicInteger or synchronized for atomicity.

What is StampedLock and how does its optimistic read mode work?

Why asked: Java 8+ lock — shows awareness of lock-striping and optimistic concurrency strategies.
💡 Hint

Optimistic read acquires a stamp without blocking. You perform the read, then call validate(stamp). If it returns false (a write occurred), fall back to a full read lock. This avoids the overhead of read-lock acquisition in low-write-contention scenarios. StampedLock is non-reentrant.

ExecutorService & ThreadPoolExecutor

5.15 🟡 Intermediate

Explain ThreadPoolExecutor parameters: corePoolSize, maximumPoolSize, keepAliveTime, and workQueue. How does the executor decide to spawn new threads?

Why asked: Critical for production systems — wrong pool configuration causes thread starvation or OOM.
💡 Hint

Order of operations: (1) if active threads < corePoolSize → create thread; (2) else enqueue task; (3) if queue full and active < maxPoolSize → create thread; (4) else apply RejectedExecutionHandler. Choosing queue type (bounded ArrayBlockingQueue vs unbounded LinkedBlockingQueue) has major impact on behavior.

Fork/Join & Atomic Classes

5.24 🔴 Advanced

💡 Hint

Fork/Join recursively splits tasks (RecursiveTask returns value, RecursiveAction doesn't). Work-stealing: idle threads steal tasks from the tail of busy threads' deques, reducing idle time and load imbalance. Optimal for divide-and-conquer CPU-bound work.

Deadlock, Livelock, Starvation & ThreadLocal

5.28 🟡 Intermediate

What are the four necessary conditions for a deadlock? How do you detect and prevent it in Java?

💡 Hint

Coffman conditions: mutual exclusion, hold-and-wait, no preemption, circular wait. Prevention: lock ordering (always acquire locks in same global order), tryLock(timeout) to back off, or use higher-level abstractions. Detection: thread dump (jstack) shows threads waiting in a cycle.

5.29 🟡 Intermediate

What is the difference between deadlock, livelock, and starvation?

Why asked: Distinguishing these concepts demonstrates real-world concurrency debugging experience.
5.30 🟡 Intermediate

What is ThreadLocal? What are common use cases and potential pitfalls (especially in thread pools)?

💡 Hint

ThreadLocal provides per-thread storage — useful for request context (user ID, transaction) without passing it through every method. In thread pools, threads are reused: always call remove() after use to prevent stale data leaking across requests. Overuse leads to hidden coupling and memory leaks.

6.6 🟢 Basic

What is a Stream in Java? How does lazy evaluation work, and what triggers terminal operations?

Java 21: Virtual Threads & Modern Features

6.26 🔴 Advanced

What are virtual threads (Project Loom, Java 21)? How do they differ from platform threads and what problem do they solve?

Why asked: Virtual threads are the most significant Java concurrency advancement in years — expected knowledge for any 2024+ senior role.
💡 Hint

JVM Architecture & Runtime Data Areas

7.1 🟡 Intermediate

Garbage Collection

7.5 🟡 Intermediate

Explain the Java heap generational structure: Eden, Survivor spaces (S0/S1), Old Gen. How does object promotion work?

Why asked: Generational GC is the basis for all tuning conversations — expected knowledge for any performance-oriented role.
💡 Hint

New objects allocated in Eden. Minor GC copies live objects to Survivor space; objects surviving multiple GCs (threshold set by -XX:MaxTenuringThreshold) get promoted to Old Gen. Major GC collects Old Gen. Large objects (humongous allocations) go directly to Old Gen / G1 humongous regions.

7.6 🟡 Intermediate

Compare GC algorithms: Serial, Parallel, CMS, G1, ZGC, and Shenandoah. When would you choose each?

7.7 🔴 Advanced

How does G1GC work? What are its regions, concurrent marking phases, and how does it achieve predictable pause times?

💡 Hint

G1 divides heap into equal-sized regions (typically 1–32 MB). It builds a "remembered set" per region to track inter-region references. Evacuation (copying) selects regions with most garbage first (hence "Garbage First"). Concurrent marking runs alongside application. Pause target set via -XX:MaxGCPauseMillis.

Why asked: ZGC is the GC of choice for latency-sensitive services — senior engineers are expected to know its trade-offs.

7.10 🟡 Intermediate

What is a memory leak in Java? Give three real-world examples and explain how to detect them.

💡 Hint

Classic examples: (1) Static collections growing unboundedly. (2) Unclosed streams/connections (pre try-with-resources). (3) Listeners/callbacks not deregistered. (4) ThreadLocal not removed in thread pools. (5) Custom classloaders preventing class unloading. Detection: heap dumps with jmap, analysis with Eclipse MAT or VisualVM.

JIT, AOT & GraalVM

7.14 🔴 Advanced

How does JIT compilation work in the JVM? What are C1 and C2 compilers and how does tiered compilation decide between them?

Why asked: JIT is why Java can match C++ performance in long-running apps — interviewers probe this for systems design awareness.
