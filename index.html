T1·Q5 Explain compile-time (static) polymorphism vs runtime (dynamic) polymorphism with concrete examples. How does JVM achieve dynamic dispatch?
🟡 Intermediate
💡 Hint & Answer Guide
Compile-time = method overloading (resolved by compiler based on parameter types). Runtime = method overriding via vtable lookup. JVM uses a virtual method table (vtable) per class; each object's reference points to its class's vtable. Invokevirtual bytecode instruction triggers dynamic dispatch. Key: final, static, and private methods are NOT dynamically dispatched — they use invokestatic or invokespecial.
T1·Q6 What is the difference between method overloading and method overriding? Can you override a static method? What about a private method?
🟢 Basic

Why asked: Surprisingly many candidates confuse method hiding (static) with overriding — a favorite gotcha question.

T1·Q7 What are the rules for valid method overriding in Java? Include covariant return types, access modifiers, and exceptions in your answer.
🟡 Intermediate
T1·Q8 Can an abstract class have a constructor? If so, when is it called and what is its purpose?
🟢 Basic
T1·Q9 What are default and static methods in interfaces (Java 8+)? What problem do they solve, and what happens when a class implements two interfaces with the same default method?
🟡 Intermediate

Why asked: Tests Java 8 knowledge and conflict resolution — a compiler error forces explicit override, which many candidates miss.

💡 Hint & Answer Guide
If two interfaces define the same default method, the implementing class MUST override it or the code won't compile. You can explicitly call a specific interface's default via InterfaceName.super.methodName(). Static interface methods are not inherited by implementing classes — they must be called via the interface name directly.

T1·Q11 What is the Open/Closed Principle? How do Strategy pattern or abstract classes help achieve OCP in Java?
🟡 Intermediate
T1·Q12 Explain the Liskov Substitution Principle with a Java example of a violation (hint: Square/Rectangle problem) and how to fix it.
🔴 Advanced
💡 Hint & Answer Guide
Classic LSP violation: Square extends Rectangle overrides setWidth/setHeight to keep sides equal, breaking callers that expect independent dimensions. Fix: don't use inheritance — make both implement a Shape interface, or use composition. LSP is violated when a subclass changes postconditions or preconditions of the superclass contract.

🟢 Basic
T1·Q18 What are sealed classes (Java 17)? How do they relate to OCP and type-safe hierarchies? Give a use case where sealed classes are superior to regular inheritance.
🔴 Advanced
💡 Hint & Answer Guide
Sealed classes (sealed class Shape permits Circle, Rectangle) restrict which classes can extend them, enabling exhaustive pattern matching in switch expressions. They allow the compiler to verify completeness in switch without a default case. Good for domain models like Result, Event, or Either types where all variants are known at compile time.

Topic 2 — Collections Framework

T2·Q1 What is the Collections framework hierarchy? Explain the relationship between Collection, List, Set, Queue, and Map.
🟢 Basic

🟢 Basic

Why asked: Fundamental data structure question — wrong choices here cause real performance issues in production code.

T2·Q3 How does ArrayList grow internally? What is its default initial capacity and growth factor? What is the cost of resizing?
🟡 Intermediate
💡 Hint & Answer Guide
Default capacity = 10. Growth = newCapacity = oldCapacity + (oldCapacity >> 1) (i.e., 1.5× growth via right-shift). Resizing involves Arrays.copyOf(), which is O(n). Use new ArrayList<>(expectedSize) to pre-size when you know the expected number of elements, avoiding repeated resizing overhead.
T2·Q4 Why is Vector considered legacy? How does it differ from ArrayList in terms of thread safety, and what is the modern alternative for thread-safe lists?
🟡 Intermediate

Why asked: Tests awareness of legacy APIs and modern concurrent alternatives like CopyOnWriteArrayList.

T2·Q5 Explain HashMap's internal structure — buckets, hash function, and how key-value pairs are stored. What happens on hash collision?
🟡 Intermediate

Why asked: One of the most common Java interview questions — deep understanding of internals separates strong candidates.

T2·Q6 What is the load factor in HashMap? What are the default values, and how does rehashing work? When does Java 8 convert a linked list bucket to a tree?
🔴 Advanced
💡 Hint & Answer Guide
Default initial capacity = 16, default load factor = 0.75. Rehashing triggers when size > capacity × loadFactor. Java 8: when a bucket's chain length exceeds TREEIFY_THRESHOLD = 8 AND total map size ≥ MIN_TREEIFY_CAPACITY = 64, the linked list is converted to a Red-Black tree (O(log n) lookup vs O(n)). Tree converts back to list when size drops below UNTREEIFY_THRESHOLD = 6.
T2·Q7 What happens if you use a mutable object as a HashMap key and then mutate it? How does this relate to the hashCode contract?
🔴 Advanced

Why asked: A subtle bug — mutating a key after insertion makes it "lost" in the map since the hash slot changes. Reveals deep understanding of map internals.

T2·Q8 Compare LinkedHashMap vs TreeMap vs HashMap — ordering guarantees, performance characteristics, and appropriate use cases.
🟡 Intermediate
T2·Q9 How does ConcurrentHashMap achieve thread safety without locking the entire map? How does it differ from Collections.synchronizedMap()?
🔴 Advanced
💡 Hint & Answer Guide
Java 8 ConcurrentHashMap uses CAS (Compare-And-Swap) operations + synchronized on individual bucket heads (not the whole map). Segment-based locking from Java 7 was removed. Default concurrency level effectively allows multiple writers on different buckets simultaneously. synchronizedMap wraps every operation in a single mutex — one lock for the entire map — creating a bottleneck. ConcurrentHashMap also has atomic compound operations: putIfAbsent, computeIfAbsent, merge.
T2·Q10 Explain the difference between HashSet, LinkedHashSet, and TreeSet. How is HashSet backed internally?
🟢 Basic

Why asked: HashSet is backed by a HashMap — knowing this reveals understanding of composition in the Collections API.

T2·Q11 What is a PriorityQueue? How does it determine ordering, and what is its time complexity for insertion and poll?

T2·Q13 What is a BlockingQueue? Name three implementations and their use cases in producer-consumer patterns.
🟡 Intermediate
T2·Q14 Explain the contract between equals() and hashCode(). What happens if you override only equals without hashCode?
🟡 Intermediate
💡 Hint & Answer Guide
Contract: if a.equals(b) then a.hashCode() == b.hashCode() (not necessarily vice versa). Violating this breaks HashMap/HashSet: two "equal" objects could hash to different buckets, so set.contains(obj) returns false even after you added an equal object. Always use Objects.equals() and Objects.hash() or IDE-generated implementations. In Java 16+, records auto-generate correct equals/hashCode.

🟢 Basic
T2·Q24 How would you implement an LRU cache in Java using LinkedHashMap? Walk through the access-order constructor and removeEldestEntry override.
🔴 Advanced
💡 Hint & Answer Guide
new LinkedHashMap<>(capacity, 0.75f, true) — the third parameter accessOrder=true re-orders on every get/put. Override removeEldestEntry: return size() > maxCapacity. Wrap in Collections.synchronizedMap() or use explicit locking for thread safety. Time complexity: O(1) get/put. Alternatively, Caffeine library provides a production-grade, thread-safe LRU/LFU cache.

Topic 3 — Generics, Enums & Annotations

T3·Q1 What are generics in Java? What problems do they solve, and how do they interact with the type system at runtime?
🟢 Basic
T3·Q2 What is type erasure? What are its implications at runtime — can you create new T() or new T[]? How does instanceof work with generics?
🔴 Advanced
💡 Hint & Answer Guide
Type erasure replaces generic type parameters with their bounds (or Object if unbounded) at compile time. At runtime, List<String> and List<Integer> are both just List. Cannot do: new T(), new T[], instanceof List<String>. Workarounds: pass a Class<T> token, use Guava's TypeToken or Jackson's TypeReference to retain type at runtime. instanceof List<?> (unbounded wildcard) IS allowed.
T3·Q3 Explain upper-bounded wildcards (? extends T) vs lower-bounded wildcards (? super T). What is the PECS principle?
🟡 Intermediate

Why asked: PECS (Producer Extends, Consumer Super) is a key API design concept — misuse leads to type-unsafe or overly restrictive method signatures.

T3·Q4 Why is List<String> not a subtype of List<Object>? What is the concept of invariance in generics, and how do arrays differ?
🟡 Intermediate
💡 Hint & Answer Guide
Generics are invariant: List<String> is NOT a List<Object> — if it were, you could add an Integer to a List<String> through the Object reference, breaking type safety. Arrays ARE covariant (String[] is an Object[]), which is why ArrayStoreException exists at runtime. Use wildcards (List<? extends Object>) for covariance.
T3·Q5 How do you write a generic method in Java? Show an example of a generic utility method with a bounded type parameter.
🟢 Basic
T3·Q6 What is heap pollution in generics? What does the @SafeVarargs annotation do and when is it safe to use it?
🔴 Advanced

Why asked: Heap pollution with varargs is a subtle type safety issue — tests deep generics knowledge beyond basic usage.

T3·Q7 Can you use generics with static fields or methods? What are the restrictions and why?
🟡 Intermediate
T3·Q8 What is an enum in Java? How is it different from a class? Can an enum have constructors, fields, and methods? Can it implement interfaces?
🟢 Basic
T3·Q9 How do you define abstract methods in an enum? Give a practical example where each enum constant provides a different implementation.
🔴 Advanced
💡 Hint & Answer Guide
Each enum constant can provide its own anonymous class body: PLUS { public int apply(int x, int y) { return x + y; } }. This is the "constant-specific method implementation" pattern from Effective Java. It's a clean alternative to switch statements on enums — behavior is encapsulated in the constant itself, following OCP. The abstract method forces every constant to provide an implementation.
T3·Q10 What are EnumSet and EnumMap? Why are they more efficient than HashSet/HashMap for enum keys?
🟡 Intermediate

Why asked: EnumSet uses a bit-vector internally for O(1) operations — a performance win that many developers miss.

T3·Q11 How do you safely get an enum constant from a string? What happens if the string doesn't match? How can you handle this gracefully?
🟢 Basic
T3·Q12 What is a Java annotation? Explain the difference between marker, single-value, and full annotations with examples.
🟢 Basic
T3·Q13 Explain @Retention policy values — SOURCE, CLASS, and RUNTIME. When would you use each?
🟡 Intermediate

Why asked: Misusing retention policy is a common mistake — if you need annotation-driven runtime behavior (like Spring), you MUST use RUNTIME.

T3·Q14 What does @Target control in an annotation definition? List the possible ElementType values.
🟢 Basic
T3·Q15 How do you create and process a custom annotation at runtime using reflection? Write a skeleton of a validation annotation processor.
🔴 Advanced
💡 Hint & Answer Guide
Define annotation with @Retention(RUNTIME) @Target(FIELD). In the processor: use clazz.getDeclaredFields(), check field.isAnnotationPresent(MyAnnotation.class), read values via field.getAnnotation(MyAnnotation.class).value(). Don't forget field.setAccessible(true) for private fields. This is exactly how Hibernate Validator, Jackson, and Spring work under the hood.
T3·Q16 What is annotation inheritance in Java? Can annotations be inherited by subclasses? What does @Inherited do?
🟡 Intermediate
T3·Q17 What are repeatable annotations (Java 8)? How do you declare them using @Repeatable?
🟡 Intermediate
T3·Q18 How do annotation processors (APT) work at compile time? Name two popular frameworks that heavily use compile-time annotation processing.
🔴 Advanced

Why asked: Compile-time annotation processing (Lombok, MapStruct) is heavily used in modern Java projects — understanding it shows framework-level depth.

T3·Q19 Explain enum's ordinal() and name() methods. Why is it dangerous to persist enums by ordinal?
🟡 Intermediate
T3·Q20 Can generics be used with exceptions? Can you have a generic catch clause like catch (T e)? What about throwing generic exceptions?
🔴 Advanced
💡 Hint & Answer Guide
You CANNOT catch a generic type (catch (T e) is a compile error) because of type erasure — the JVM can't distinguish types at runtime in catch blocks. However, you CAN declare generic throws: <T extends Exception> void execute(Task<T> task) throws T. This allows callers to catch the specific exception type. Used in libraries like Guava's Throwables.propagateIfPossible.

Topic 4 — Exception Handling

T4·Q1 What is the exception class hierarchy in Java? Draw the relationship between Throwable, Error, Exception, RuntimeException, and checked exceptions.
🟢 Basic
T4·Q2 What is the difference between checked and unchecked exceptions? Give examples of each, and explain the philosophical debate around checked exceptions.
🟢 Basic

Why asked: The "should we use checked exceptions" debate is ongoing — candidates who can articulate both sides show real experience.

T4·Q3 What is try-with-resources? How does it work internally, and what interface must a resource implement?
🟢 Basic
T4·Q4 What are suppressed exceptions in try-with-resources? How do you access them, and when are they created?
🔴 Advanced
💡 Hint & Answer Guide
If the try block throws AND close() also throws, the close-exception is suppressed to preserve the original exception. Access suppressed exceptions via e.getSuppressed(). Prior to try-with-resources (Java 7), developers had to manually suppress close exceptions using nested try-finally, often accidentally losing the original exception. This was a very real bug in many codebases.
T4·Q5 What is multi-catch syntax in Java 7+? What restriction exists on the exception variable in multi-catch, and why?
🟢 Basic
T4·Q6 When is the finally block NOT executed? Name at least three scenarios where finally is skipped.
🟡 Intermediate

Why asked: Most candidates say "finally always runs" — but System.exit(), JVM crash, infinite loop, and thread kill are edge cases that trip up assumptions.

T4·Q7 What value is returned from a method when both the try block and the finally block have return statements? What does the JVM do?
🔴 Advanced
💡 Hint & Answer Guide
The finally block's return OVERRIDES the try block's return — the try block's return value is silently discarded. This is considered bad practice. Similarly, if finally throws an exception, it overrides any exception from the try block. This is why returning from or throwing in finally blocks is a code smell flagged by static analysis tools like SonarQube.
T4·Q8 What are best practices for creating custom exceptions? When should you extend RuntimeException vs Exception?
🟡 Intermediate

Why asked: Custom exception design directly impacts how usable and maintainable an API is — a common real-world design decision.

T4·Q9 How do you re-throw an exception? What is the difference between throw e and throw new WrapperException(e)? How do you preserve the stack trace?
🟡 Intermediate
T4·Q10 What is exception chaining? How does passing the original exception as a cause to a new exception help with debugging?
🟡 Intermediate
T4·Q11 Should you catch Exception or Throwable? What are the risks of each, and when might catching Throwable be justified?
🟡 Intermediate

Why asked: Catching Throwable includes Errors like OutOfMemoryError — doing so can mask JVM-level failures and prevent proper cleanup.

T4·Q12 What is the difference between StackOverflowError and OutOfMemoryError? Are they recoverable?
🟢 Basic
T4·Q13 How do you handle exceptions in lambdas and streams? Why can't you throw checked exceptions directly inside a lambda passed to forEach?
🔴 Advanced
💡 Hint & Answer Guide
Consumer<T>'s accept method doesn't declare checked exceptions, so lambdas using it can't throw checked exceptions. Solutions: (1) wrap in try-catch inside the lambda, (2) create a functional interface that throws checked exceptions (@FunctionalInterface ThrowingConsumer), (3) use Lombok's @SneakyThrows, or (4) use Vavr's CheckedFunction. The cleanest approach for production is a custom wrapper function.
T4·Q14 What is an UncaughtExceptionHandler? How do you set one for a thread or for all threads globally?
🔴 Advanced
T4·Q15 How do you implement a global exception handler in a Spring Boot application using @ControllerAdvice? How does it relate to Java's exception handling model?
🟡 Intermediate

Why asked: Bridges pure Java and Spring — tests practical experience in converting exceptions to meaningful HTTP responses.

T4·Q16 What are the performance implications of exceptions? Should you use exceptions for flow control? Why or why not?
🟡 Intermediate
T4·Q17 What does the Closeable interface add over AutoCloseable? Can a resource's close() method throw a checked exception in try-with-resources?
🟡 Intermediate
T4·Q18 How do you write an exception-safe resource cleanup in Java without try-with-resources (e.g., for a legacy resource that doesn't implement AutoCloseable)?
🔴 Advanced
T4·Q19 What is the difference between NoSuchMethodException, NoSuchMethodError, and UnsupportedOperationException? When does each occur?
🟡 Intermediate

Why asked: Distinguishing between reflection-time, link-time, and runtime failures shows understanding of the JVM's class loading and verification phases.

T4·Q20 Design an exception strategy for a layered application (Controller → Service → Repository). What exception types belong in each layer and how should translation work?
🔴 Advanced
💡 Hint & Answer Guide
Repository layer: throw DataAccessException (Spring wraps SQL exceptions). Service layer: translate to domain exceptions (UserNotFoundException, InsufficientFundsException). Controller layer: @ControllerAdvice maps domain exceptions to HTTP status codes. Never expose infrastructure exceptions (SQLException, HibernateException) to the presentation layer — this leaks implementation details. Use distinct exception hierarchies per bounded context.
